# 硬性规则（每次会话必须遵守）

本工程要上传 GitHub，以下两条是不可违反的红线。每次打开 Claude 都应先了解：

## 规则 1：禁止品牌字眼

- 代码、注释、文档、标识符、提交信息中**不得出现**参考工程相关的品牌字眼。
- **具体禁用词清单写在 `.claude/local-notes.md`（已 gitignore，不入库）**——
  本文件会被提交，所以这里不把那些词写出来，只指代。
- 参考工程一律称为 **`agent-server`**；它的本地真实路径同样只在 `.claude/local-notes.md`。
- 例外：vendored 第三方代码（langchaingo）里碰巧含这些子串的普通单词不算违规，
  不要去改 vendored 代码。

## 规则 2：禁止硬编码密钥

- 代码中**不得出现**任何 `token`、`api-key`、密钥、口令等敏感信息。
- 所有凭据只通过**环境变量**注入，模板见 `.env.example`。
- 运行期由 `domain/account` 负责加载凭据，代码里只引用变量名，绝不写死值。
